# เอกสารการพัฒนาระบบ (Development)
## โครงงาน: BM Computer (บ้านมีคอม)

เอกสารนี้อธิบายกระบวนการพัฒนาระบบจริง: แนวทาง เครื่องมือ โครงสร้างโค้ด
การทำงานเป็นทีมด้วย Git/GitHub และเหตุการณ์สำคัญ (milestones) ระหว่างการพัฒนา

---

## 1. แนวทางการพัฒนา (Iterative Development)

ทีมพัฒนาแบบ **ทำเป็นรอบ (Iterative)**: แต่ละรอบเลือกฟีเจอร์มาทำให้ **สมบูรณ์จริงทั้งชิ้น**
(หน้าเว็บ + API + ฐานข้อมูล + ทดสอบ) ก่อนขึ้นรอบถัดไป ภายใต้กติกาของทีม:

1. **ห้ามทำครึ่งๆ กลางๆ** - ทุกปุ่มที่ขึ้นหน้าเว็บต้องทำงานจริง (เพิ่มตะกร้า = เพิ่มจริง, จ่ายเงิน = เกิดออเดอร์จริง)
2. **Dynamic ทั้งเว็บ** - ห้าม hardcode สินค้า/หมวด/เนื้อหาในโค้ด ทุกอย่างต้องมาจากฐานข้อมูลและคุมจากหลังบ้าน
3. **ลำดับความสำคัญ**: ความปลอดภัย > ประสิทธิภาพ > ประสบการณ์ผู้ใช้
4. **ทุกข้อความ 2 ภาษา** ผ่านระบบแปลกลาง และธีมสี dark/light ผ่าน design tokens

## 2. เครื่องมือที่ใช้พัฒนา

| เครื่องมือ | ใช้ทำอะไร |
|-----------|-----------|
| VS Code | เขียนโค้ดหลักของทีม |
| Vite dev server + Wrangler dev | รันหน้าเว็บ (:5173) และ Backend (:8787) บนเครื่อง พร้อม hot reload |
| Git + GitHub | ควบคุมเวอร์ชันโค้ด แบ่งงาน ตรวจงานร่วมกัน |
| Cloudflare Pages / Workers | Deploy อัตโนมัติเมื่อ push ขึ้น branch `main` (CI/CD) |
| Supabase Dashboard + Migrations | จัดการฐานข้อมูล สคีมาเก็บเป็นไฟล์ migration ทุกครั้งที่แก้ |
| Swagger UI (`/api/docs`) | เอกสาร API อัตโนมัติ + ใช้ทดสอบยิง API ระหว่างพัฒนา |
| เบราว์เซอร์ DevTools | ตรวจ network/console/responsive ทุกฟีเจอร์ก่อนถือว่าเสร็จ |

## 3. โครงสร้างโค้ด

```
bm-computer/
├── src/                     # Frontend (React 18 + Vite)
│   ├── components/          # ชิ้นส่วน UI ใช้ซ้ำ (Navbar, ProductCard, Carousel, Skeleton ฯลฯ)
│   ├── pages/               # หน้าเว็บทั้งหมด (Home, ProductList, Checkout, PCBuilder ฯลฯ)
│   │   ├── account/         # โซนบัญชีของฉัน (โปรไฟล์ ที่อยู่ ภาษี wishlist)
│   │   ├── admin/           # หลังบ้านทั้งระบบ (ภาพรวม สินค้า ออเดอร์ ชำระเงิน ฯลฯ)
│   │   └── docs/            # หน้าเอกสารระบบ (/docs)
│   ├── lib/                 # ชั้นคุยกับ API, ค้นหา fuzzy, PromptPay QR, PC Builder engine
│   ├── auth/ cart/ catalog/ wishlist/  # ตัวจัดการสถานะ (React Context)
│   ├── i18n/                # ระบบ 2 ภาษา - ข้อความทั้งหมดอยู่ที่นี่
│   └── theme/               # Dark/Light mode
├── backend/                 # Backend API (Cloudflare Worker: Hono + OpenAPI)
│   └── src/modules/         # auth · catalog · orders · payments · account · builder · admin
├── supabase/                # schema.sql + migrations/ (ประวัติการแก้ฐานข้อมูลทั้งหมด)
├── functions/               # Pages Function: proxy /api/* ให้คุกกี้เป็น first-party
└── docs/                    # เอกสารโครงงาน (ไฟล์นี้และเพื่อน)
```

## 4. การทำงานเป็นทีมด้วย Git / GitHub

- Repository: `github.com/manatsawintho-ragoon/bm-computer`
- แบ่งงานเป็นฟีเจอร์ ทำเสร็จแล้ว commit พร้อมข้อความอธิบายชัดเจน (ภาษาไทย/อังกฤษ)
- **push ขึ้น `main` = ระบบ deploy อัตโนมัติ** ทั้งเว็บ (Cloudflare Pages) และ API (Worker)
  ทำให้ทั้งทีมเห็นผลงานบนเว็บจริงทันทีทุกครั้ง
- การแก้ฐานข้อมูลทุกครั้งเก็บเป็นไฟล์ `supabase/migrations/<เวลา>_<ชื่องาน>.sql`
  ย้อนดูได้ว่าใครแก้อะไรเมื่อไร และติดตั้งซ้ำบนโปรเจกต์ใหม่ได้
- สมาชิกทุกคน clone แล้วรันได้เองด้วยคู่มือ [`SETUP.md`](../SETUP.md) (ต่อฐานข้อมูลก้อนเดียวกันทั้งทีม)

## 5. เหตุการณ์สำคัญระหว่างพัฒนา (Milestones)

| รอบ | งานที่ส่งมอบ |
|-----|-------------|
| 1 | โครงหน้าเว็บ + ธีมแดง-ขาว-เทา + หน้าแรก dynamic (สไลด์/Flash Sale/แบรนด์จริงจากฐานข้อมูล) |
| 2 | ระบบสมาชิก + ตะกร้าจริง + checkout สร้างออเดอร์จริง + ติดตาม/ประวัติออเดอร์ |
| 3 | Admin CMS: สินค้า/หมวด/แบรนด์/สไลด์/ออเดอร์ (CRUD ครบ เขียนฐานข้อมูลจริง) |
| 4 | แยก Backend API (Cloudflare Worker + Swagger 40+ เส้น) + session HttpOnly หมดอายุจริง + Google OAuth |
| 5 | PromptPay QR ล็อกยอด + ตรวจสลิป EasySlip + ตัด/คืนสต็อกอัตโนมัติ + ยกเลิก/คืนเงินครบวงจร |
| 6 | PC Builder เต็มระบบ (ตรวจความเข้ากันได้ 19 กฎ) + สเปคชุมชน + รีวิวผู้ซื้อจริง |
| 7 | บัญชีของฉัน (ที่อยู่/ภาษี/ชำระเงิน/wishlist) + ค้นหา autocomplete + สินค้า 150+ ชิ้น 39 แบรนด์ |
| 8 | ปรับปรุงรอบสุดท้าย: หน้าตัวกรองสินค้าใหม่ · แดชบอร์ดกราฟ + ตัวกรองหลังบ้าน · สมัครสมาชิกไม่ต้องยืนยันอีเมล · รองรับ localhost เต็มรูปแบบ · หน้าเอกสารระบบ /docs |

## 6. ปัญหาที่พบและการแก้ไข (ตัวอย่างจริง)

| ปัญหา | สาเหตุ | วิธีแก้ |
|-------|--------|--------|
| Google login สำเร็จแต่ระบบเหมือนไม่ล็อกอิน | หลังโอน session เข้า cookie แล้วเรียก signOut ฝั่ง client ซึ่งไปเพิกถอน session ที่เซิร์ฟเวอร์ด้วย | ตั้ง client เป็นทางผ่าน OAuth เท่านั้น (ไม่เก็บ session ไม่ signOut) - session อยู่ที่ cookie ที่เดียว |
| เบราว์เซอร์บางตัวบล็อกคุกกี้ (ข้ามโดเมน pages.dev → workers.dev) | คุกกี้ third-party ถูกบล็อกมากขึ้นทุกเบราว์เซอร์ | เพิ่ม proxy ฝั่ง Pages ให้เรียก API แบบโดเมนเดียวกัน คุกกี้จึงเป็น first-party |
| กดการ์ดสินค้าใน carousel หน้าแรกไม่ได้ | การจับ pointer เพื่อรองรับการ "ลาก" ไปดักเหตุการณ์คลิกของการ์ด | จับ pointer เฉพาะเมื่อผู้ใช้เริ่มลากจริง (ขยับเกิน 3px) คลิกปกติจึงทะลุถึงการ์ด |
| สมัครสมาชิกแล้วเข้าระบบไม่ได้ | โปรเจกต์เปิด "ยืนยันอีเมล" ไว้ ผู้ใช้ไม่ได้กดลิงก์ในเมล | Backend ยืนยันอีเมลให้อัตโนมัติแล้วพาเข้าสู่ระบบทันทีหลังสมัคร |
| อีเมลสมัครพิมพ์ผิด (เช่น gamil.com) ทำเมลตีกลับ | ผู้ใช้พิมพ์โดเมนผิด | ระบบเช็คโดเมน + เสนอคำแก้ให้กดยืนยันก่อนสมัคร |

## 7. มาตรฐานคุณภาพที่ยึดระหว่างพัฒนา

- ตรวจ `npm run build` และ TypeScript (`tsc --noEmit`) ผ่านก่อน push ทุกครั้ง
- ทุกฟีเจอร์ทดสอบกับฐานข้อมูลจริงก่อนถือว่าเสร็จ (ดูผลใน [testing.md](./testing.md))
- ข้อความ error ทุกจุดเป็นภาษาคน อ่านแล้วรู้ว่าต้องทำอะไรต่อ
- โหลดช้าให้เห็น skeleton ไม่ใช่จอขาว · ทุก action มี feedback (สำเร็จ/ล้มเหลว)
